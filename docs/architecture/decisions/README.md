# Architecture Decisions

Record architecture decision records in this folder when a change affects deployment boundaries, security posture, provider contracts, state layout, approval behavior, or operator workflow.

Recommended ADR format:

```text
# ADR-0001: Short Decision Title

## Status

Proposed | Accepted | Superseded

## Context

What forces, constraints, and alternatives matter?

## Decision

What are we choosing?

## Consequences

What gets easier, harder, safer, or riskier?
```

Start a new ADR before changing public contracts, generated artifact shape, apply semantics, secret handling, or dashboard authorization behavior.
